## 1. ビジョンと戦略

### 1-1. ビジョン

> **WALLERは、ウォールトランポリンの挑戦と成長を世界中で共有し、応援しあえるプラットフォームである。
> マイナー競技の普及を加速し、競技団体設立とオリンピック競技化への礎を築く。**

### 1-2. ミッション

- ウォールトランポリンの「挑戦」を気軽にシェアし、応援される体験を作る
- 世界のプレイヤーがつながれる場を提供する
- 技の体系化・プレイヤー数の可視化を通じて、競技の普及基盤となる

### 1-3. 戦略的ポジショニング

**インスタグラムとの差別化:**

| | Instagram | WALLER |
|---|---|---|
| コンテンツ | 映える完成品 | 練習も失敗もシェア |
| リアクション | いいね（上手い人が勝つ） | 拍手（挑戦に送る） |
| 価値 | フォロワー数 | 成長記録・技の蓄積 |
| 技の扱い | ハッシュタグ（構造なし） | 技辞典（体系化） |
| 施設連携 | なし | 施設ページ・MAP |
| 競技データ | なし | プレイヤー数・技数・国別分布 |

**WALLERでしかできないこと:**
- 技タグで投稿が技辞典に自動蓄積される
- 自分の成長が技別・時系列で見える
- 同じ施設で練習する仲間の投稿が見える
- 競技全体のデータが可視化される

### 1-4. コミュニティ方針

ウォールトランポリンの現在の文化的資産は「暖かいコミュニティ」であり、これを最優先で守る。

**守るべき空気感:**
- 挑戦に拍手を送る文化
- 初心者も萎縮せず投稿できる環境
- レベルに関係なく応援される場

**やらないこと:**
- 難易度ランキング・スコアリング（萎縮の元）
- 上手さで序列をつける仕組み
- 映え重視の文化を助長するUI

## 3. デザインシステム

### 3-1. デザイン方針

ミニマル・モノトーン。挿し色にオレンジ。
余白を多くとり、情報を詰め込みすぎない。ゲーム的なクールさとクリーンさの両立。

### 3-2. カラーパレット

#### ライトモード

| 用途 | カラー | コード |
|---|---|---|
| 背景（メイン） | 白 | `#FFFFFF` |
| 背景（カード・セクション） | 薄グレー | `#F5F5F5` |
| テキスト（メイン） | ほぼ黒 | `#1A1A1A` |
| テキスト（サブ） | グレー | `#6B6B6B` |
| ボーダー・区切り | 薄グレー | `#E5E5E5` |
| ボタン（プライマリ） | 黒 | `#1A1A1A` |
| ボタン（セカンダリ） | 薄グレー | `#F5F5F5` |
| アクセント | オレンジ | `#FF8C00` |
| エラー | 赤 | `#DC3545` |
| 成功 | 緑 | `#28A745` |

#### ダークモード（Phase 2以降）

| 用途 | カラー | コード |
|---|---|---|
| 背景（メイン） | ほぼ黒 | `#0A0A0A` |
| 背景（カード） | ダークグレー | `#1A1A1A` |
| テキスト（メイン） | オフホワイト | `#F5F5F5` |
| テキスト（サブ） | グレー | `#999999` |
| アクセント | 明るめオレンジ | `#FFB800` |

#### オレンジの使用ルール

オレンジは「嬉しいこと・アクション」のシグナルとして厳選使用:
- 拍手アイコン・カウント
- 投稿ボタン（+）
- アクティブタブの下線/アイコン
- 「できた」ムードタグ
- 通知バッジ

**それ以外は全てモノトーン。** オレンジが目に入る = 何か嬉しいことが起きている。

### 3-3. タイポグラフィ

| 要素 | フォント | サイズ | ウェイト | 備考 |
|---|---|---|---|---|
| 見出し | SF Pro / Inter | 20-24px | Bold | letter-spacing: -0.5px |
| 本文 | SF Pro / Inter | 14-16px | Regular | |
| サブテキスト | SF Pro / Inter | 12-13px | Regular | #6B6B6B |
| ボタン | SF Pro / Inter | 15px | Semibold | |
| 数値 | SF Pro / Inter | - | Bold | モノスペース的ウェイト |
| ロゴ | カスタム or SF Pro | - | Black | 「WALLER」テキストロゴ |

フォント: iOS = SF Pro Display/Text、Android/Web = Inter

### 3-4. コンポーネントスタイル

| コンポーネント | スタイル |
|---|---|
| カード | 背景 #F5F5F5, 角丸 16px, padding 16px, シャドウなし |
| ボタン（黒） | 背景 #1A1A1A, テキスト白, 角丸 12px, 高さ 48px |
| ボタン（白） | 背景 #F5F5F5, テキスト黒, 角丸 12px, 高さ 48px |
| タグ（pill） | 背景 #F5F5F5, 角丸 20px, padding 6px 14px |

| 入力フィールド | 背景 #F5F5F5, 角丸 12px, ボーダーなし, padding 12px 16px |

### 3-5. アイコン

- スタイル: ライン（塗りなし）、ストローク2px
- カラー: #1A1A1A（通常）、#FF8C00（アクティブ/アクセント）
- 推奨ライブラリ: Lucide Icons or Phosphor Icons
- サイズ: 24px（タブバー）、20px（インライン）

## 4. 機能仕様

### 4-1. クリップ投稿

#### 動画仕様

| 項目 | 仕様 | 理由 |
|---|---|---|
| 最大尺 | 15秒 | ストレージコスト削減、技の瞬間に特化 |
| 最小尺 | 1秒 | 極端に短い投稿を防止 |
| 最大サイズ | 50MB（圧縮前） | 圧縮後は5-8MB想定 |
| 解像度上限 | 720p | コスト最適化 |
| 形式 | MP4, MOV（H.264） | |
| 圧縮 | クライアント側自動圧縮 | |
| サムネイル | 動画1秒目を自動抽出 | |

#### 投稿入力項目

| 項目 | 必須 | 仕様 |
|---|---|---|
| 動画 | 必須 | カメラロールから選択、15秒以内にトリミング |
| ムードタグ | 必須 | 1つ選択（下記参照） |
| 技タグ | 任意 | 技辞典から検索 or 新規登録、複数可 |
| 施設タグ | 任意 | 施設一覧から検索 or フリー入力 |
| キャプション | 任意 | 200文字まで |

#### ムードタグ

投稿の文脈を伝えるもの。必須にすることで「何のための投稿か」が明確になり、失敗動画も「挑戦中」として自然にシェアできる。

| タグ | 用途 |
|---|---|
| 挑戦中 | 新しい技に挑戦している（失敗含む） |
| できた | 技が成功した、メイクした |
| 練習中 | 日々の練習の記録 |
| 魅せ技 | 得意技の披露 |
| はじめて | 初めての挑戦（初心者歓迎の空気作り） |

#### 投稿カード表示

```
┌──────────────────────────┐
│ @taku  ·  Trampo Lab      │  ユーザー名 / 施設名
│                          │
│ ┌────────────────────┐   │
│ │                    │   │
│ │     15秒動画        │   │
│ │                    │   │
│ │          0:12      │   │  右下に残り時間
│ └────────────────────┘   │
│                          │
│ [挑戦中] [#バックフリップ]  │  ムード / 技タグ（pill型）
│                          │
│ 3回目でやっと成功！        │  キャプション
│                          │
│ 👏 24               ···  │  拍手カウント / メニュー
└──────────────────────────┘
```

#### 投稿後の編集・削除

| 操作 | 仕様 |
|---|---|
| 編集 | キャプション・ムード・技タグ・施設タグのみ。動画は不可 |
| 削除 | 投稿後10分以内: 制限なし。10分経過後: 1日3回まで |

#### ベストプレイ枠（プロフィール動画）

大会演技やルーティン等の長尺動画に対応するため、プロフィールに「BEST PLAY」枠を設ける。
外部リンクではなくアプリ内にアップロードし、ネイティブ再生でシームレスな体験を提供する。

| 項目 | 仕様 |
|---|---|
| 枠数 | 1ユーザーあたり最大3本 |
| 最大尺 | 60秒 |
| 最大サイズ | 100MB（圧縮前）、圧縮後 20-30MB想定 |
| 解像度上限 | 720p |
| 入れ替え | 可能（古い動画はストレージから削除） |
| ストレージ上限 | 1ユーザーあたり最大約90MB（固定上限） |

**コストの特徴:** ベストプレイはフィード投稿と異なり「蓄積」ではなく「上限」。ユーザー数に比例するだけで、時間経過で無限に増えない。

| ユーザー数 | ベストプレイ最大ストレージ | 備考 |
|---|---|---|
| 100人 | 最大 9GB | 実際は半分以下 |
| 1,000人 | 最大 90GB | 固定上限 |
| 10,000人 | 最大 900GB | 固定上限 |

**プロフィールでの表示:**

```
BEST PLAY ────────────────
┌────────┐ ┌────────┐ ┌────────┐
│  60秒   │ │  60秒   │ │  空き   │
│ 大会演技 │ │ベストコンボ│ │  ＋追加  │
└────────┘ └────────┘ └────────┘
```

ベストプレイにもムードタグ・技タグ・施設タグを付けられる。ただしフィードには流れず、プロフィール訪問時のみ閲覧可能。

### 4-3. リアクション（拍手）

#### 仕様

| 項目 | 仕様 |
|---|---|
| リアクション種類 | 拍手（👏）のみ |
| 操作 | タップで1拍手、連打で拍手数が増加（最大10） |
| 表示 | 「👏 24」のように合計数を表示 |
| アニメーション | 拍手時にオレンジのパーティクル |
| 取り消し | 拍手後に再タップで取り消し |

#### 旧スタンプシステムとの違い

v0.4の4種スタンプ（火・拍手・キラキラ・筋肉）+ いいねを廃止し、**拍手1種に統一。**

理由:
- ウォールトランポリンの「挑戦に拍手を送る」文化を反映
- 種類が多いとどれを選ぶか迷う
- 拍手は上手さではなく挑戦への応援として機能する
- 連打で熱量を表現できる（ゲーム的で楽しい）

### 4-4. 技辞典

#### 概要

投稿時の技タグから自動的に技辞典が育つ仕組み。プレイヤーの学習ツールとして機能し、同時に競技の技体系化に貢献する。

#### 技の登録フロー

```
投稿時に技タグを入力
  ↓
既存技を検索 → あれば選択
  ↓
なければ「新しい技を登録」
  ↓
技名（オリジナル言語 + 英語名）を入力
  ↓
カテゴリ選択（フリップ系 / ツイスト系 / コンボ / オリジナル / その他）
  ↓
技辞典に追加、命名者として記録
```

```

│                          │
│ カテゴリ:                 │

│                          │

│ │                        ││
│ │ 投稿数: 342             ││
│ │ 挑戦者: 89人            ││
│ │ ┌────┐┌────┐┌────┐    ││
│ │ │    ││    ││    │    ││  みんなの動画
│ │ └────┘└────┘└────┘    ││
│ └────────────────────────┘│
```

#### 技とプレイヤーの紐づけ

プレイヤーのプロフィールに技の習得状況が自動反映:
- 「できた」ムードで投稿 → 習得済み
- 「挑戦中」「練習中」ムードで投稿 → 挑戦中

※ 難易度スコアは付けない。代わりに「投稿数」「挑戦者数」が自然な指標になる。

### 4-7. コメント（Phase 2）

- シンプルなテキストコメント

### 4-11. Web版（閲覧専用）

アカウント登録なしでWALLERのコンテンツを閲覧できるWeb版。
アプリの「ショーケース」として機能し、新規ユーザー獲得の入口となる。

**目的:**
- プレイヤーがSNS（LINE/X/Instagram）でクリップのリンクをシェアできる

- SEOでウォールトランポリン関連の検索を押さえる
- 「観る → 興味を持つ → アプリDL」の導線を作る

## 7. 多言語対応

### 7-1. 対応言語

| Phase | 言語 | 備考 |
|---|---|---|
| Phase 1 | 日本語、英語 | 最初から両言語で開発。デバイス言語で自動切替 |
| Phase 4+ | 需要に応じて追加 | i18n基盤があるため追加は容易 |

世界展開を見据え、**Phase 1から日英両対応を前提として開発する。** 後からi18nを追加する方がコストが高いため、最初から組み込む。

### 7-2. 実装方針

- UIテキストは**全てi18nキーで管理**（ハードコードの日本語・英語を一切排除）
- react-i18next + expo-localization
- デバイスの言語設定を自動検出、手動切替も可
- 翻訳ファイルは `src/i18n/locales/{ja,en}.json` で管理

```typescript
// 例
t('clip.mood.challenging')  // ja: "挑戦中" / en: "Challenging"
t('feed.empty')             // ja: "まだクリップがありません" / en: "No clips yet"
```

### 7-3. 技名の多言語対応

技はオリジナル言語名 + 英語名を保持。英語名がグローバル共通キーとなる。

```
日本語ユーザーの表示:  バックフリップ (Backflip)
英語ユーザーの表示:    Backflip
```

新技登録時:
- オリジナル言語名（必須）
- 英語名（推奨。未入力の場合はオリジナル名をそのまま使用）

### 7-4. ユーザー生成コンテンツ

キャプション・自己紹介等のユーザー入力テキストは翻訳しない。
施設名・チーム名も翻訳不要（エイリアスで対応可能）。

### 9-2. 収益ロードマップ

#### Phase 1-2: 自腹運営期（~300人）

月額 ~¥8,000。個人で十分持続可能。

#### Phase 2-3: 初期収益化（300~1,000人）

施設課金を待たずに運営費をカバーする手段:

| 収益源 | 内容 | 期待収益 |
|---|---|---|
| **助成金** | スポーツ庁、toto助成、笹川スポーツ財団、自治体スポーツ振興補助等。WALLERを「競技普及プロジェクト」として申請 | 数十万~数百万/年 |
| **施設スポンサー** | トランポリン施設からの少額協賛。アプリ内での施設露出を見返りに（施設課金の布石） | 月¥20,000-50,000/社 |
| **スポーツ用品メーカー協賛** | フィード内バナー、大会協賛。ニッチだがコミュニティの濃さが価値 | 月¥10,000-30,000/社 |

#### Phase 3: コミュニティ課金（1,000~3,000人）

「儲ける」ではなく「運営を支える」目的:

| 収益源 | 単価 | 想定収益 | 内容 |
|---|---|---|---|
| **サポーター制度** | 月¥300-500 | 50人参加で月¥15,000-25,000 | WALLERを支える人として名前を掲載。共感ベース |
| **プロフィールカスタマイズ** | 月¥200 | 100人購入で月¥20,000 | ロビー風背景の変更、バッジ色カスタマイズ等 |
| **拍手エフェクト** | 買い切り¥100-300 | - | 特別な拍手アニメーション |
| **オンライン大会** | 参加費¥500-1,000 | 50人参加で¥25,000-50,000/回 | WALLER杯。月1回 or 季節ごと |

#### Phase 3: 施設アカウント課金（1,000~5,000人）

| プラン | 内容 | 月額 |
|---|---|---|
| 無料 | 基本ページ（名前・場所のみ） | ¥0 |
| 有料 | 詳細ページ、プロモ投稿、イベント告知、アナリティクス | 月額（要検討） |

施設にとっての価値:
- プレイヤーが施設タグを付ける → 実質口コミ
- 施設ページに情報掲載 → 集客チャネル
- イベント告知 → 参加者獲得
- アナリティクス → 認知度の計測

#### Phase 4+: 本格収益化（5,000人~）

| 収益源 | 内容 |
|---|---|
| グッズ販売手数料 | 外部連携（SUZURI/BASE等）or マーケットプレイス |
| 大会エントリー手数料 | オンライン大会プラットフォーム |
| プレミアム機能 | プロフィールカスタマイズ拡張等 |

#### 将来: 団体化（5,000人~）

WALLERの運営を「ウォールトランポリン協会（仮）」として団体化。

| 会員種別 | 年会費 | 特典 |
|---|---|---|
| 選手会員 | ¥1,000-3,000/年 | 公式大会参加資格、選手登録 |
| 一般会員 | 無料 | アプリ利用のみ |
| 施設会員 | ¥10,000-30,000/年 | 施設ページ、イベント機能 |

1,000人の選手会員 x ¥2,000 = 年間¥2,000,000（月¥166,000）で自立運営が可能。

#### スポンサー連携

| スポンサー候補 | 見返り |
|---|---|
| トランポリン施設 | アプリ内露出 |
| スポーツ用品メーカー | フィードバナー、大会協賛 |
| 地域企業 | 大会・イベントでのロゴ掲載 |
| トランポリンパーク運営会社 | 「Powered by ○○」表示 |

### 9-3. 収益タイムライン概要

```
~300人     自腹（月~¥8,000）
  │
  │         Cloudflare R2に移行
  │
~1,000人   自腹 + 初期スポンサー1-2社 + 助成金申請
  │

~3,000人   スポンサー + サポーター + 大会で運営費カバー

  │

```
